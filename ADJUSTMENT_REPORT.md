# 調整レポート

## 2026-06-19: 初回ルールベース戦略の実装

### 変更内容
- エネルギー付与の優先順位を実装
  - 優先順位（最優先→低優先）: Mega Lucario ex > Riolu > Hariyama > Makuhita > Solrock > Lunatone
  - 各ポケモンの最大エネルギー数: Makuhita=3, Hariyama=3, Riolu=2, Mega Lucario ex=2, Solrock=1, Lunatone=2
  - 場のポケモン全員をチェックし、上限未満かつ優先度が最も高いポケモンに優先的にエネルギーを付ける
- ルナトーン（Lunar Cycle）の特性使用ロジック
  - 条件: 場にソルロックがいる & 手札に基本闘エネルギーがある & Riolu/Mega Lucario ex が場・手札にいない
  - 効果: 手札の基本闘エネルギー1枚をトラッシュして3枚ドロー（ルカリオを引きに行く）
  - 使用制限: 1ターンに1回のみ
- ハリテヤマ（Heave-Ho Catcher）の特性使用ロジック
  - 条件: ハリテヤマへの進化時、相手のベンチにハリテヤマ（Wild Press 210ダメージ）で倒せるポケモンがいる
  - 効果: 相手のベンチポケモンをバトル場に引きずり出し、倒す
- デッキアウト対策（前回から継続）
  - 山札5枚以下で type=12（ターン終了）を選択

### 変更内容（追加）
- 手札からカードをプレイする優先順位を実装:
  1. グッズ（Item）とポケモンのどうぐ（Tool）を最優先で使う
  2. リーリエの決心（Lillie's Determination）を次に使う
  3. その他のサポート（Supporter）を使う
  - サポートは1ターンに1枚のみ（`supporterPlayed` フラグで管理）
- 毎ターン必ず1回エネルギーを付けるルールに変更
  - `energyAttached` が False の場合、`type=14` があれば必ず選択
  - どのポケモンに付けるかはバトル場のポケモンで決まる（type=14はバトル場に付与）
  - 優先順位の高いポケモンをバトル場に維持するよう他ルールと連携
- 基本ポケモンをベンチに出すルールを追加
  - ベンチに空きがあり、手札に基本ポケモン（Makuhita, Lunatone, Solrock, Riolu）がある場合、type=8でプレイ
  - プレイ後の sub-selection（context=3）でベンチ位置を選択
- 進化のルールを追加
  - 手札に進化カード（Hariyama→Makuhita, Mega Lucario ex→Riolu）があり、対応するポケモンが場にいる場合、type=8で進化
- 攻撃のルールを追加
  - type=13 のオプションから最もダメージの高い攻撃を選択
  - 攻撃ダメージ: Mega Brave=270, Wild Press=210, Aura Jab=130, Cosmic Beam=70, Power Gem=50, Accelerating Stab/Confront=30, Corkscrew Punch=10
- にげるのルールを追加
  - バトル場のポケモンのHPが50以下、またはエネルギーが0の場合、type=7でにげる
- `cardType` 定数をカードデータベースの値に修正:
  - 0=Pokémon, 1=Item, 2=Tool, 3=Supporter, 4=Stadium, 5=Energy
- サブセレクション処理を汎用化: `_handle_subselect()` で context=3,7,8 を処理

### 注意点
- `type=14`（エネルギー付ける）はバトル場のポケモンにしか付けられない。そのため、優先順位の高いポケモンがベンチにいる場合は直接エネルギーを付けられない
  - 優先度判定はバトル場のポケモンのみを対象とする
- Heave-Ho Catcher は進化時にトリガーする特性で、select.type=0 の type=10 オプションとしては出現しない可能性が高い（未確認）
- ルナトーンの Lunar Cycle は実際には「手札の基本闘エネルギーを1枚トラッシュして3枚ドロー」という効果で、ルカリオをサーチするわけではない。ルカリオを引く確率を上げるために使用
